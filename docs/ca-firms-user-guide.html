1. Introduction

What is AgenticOrg CA Pack?

The AgenticOrg Chartered Accountant Firm Pack is a paid add-on for the AgenticOrg platform that provides a complete automation suite purpose-built for Indian CA firms. It deploys five AI-powered virtual employees (agents) that handle GST filing, TDS compliance, bank reconciliation, month-end close, and accounts receivable collections — across all of your client companies from a single dashboard.

Each agent operates within strict guardrails defined by Indian statutory requirements (GST Act 2017, Income Tax Act 1961, Companies Act 2013, ICAI Standards). Human-in-the-loop (HITL) review is always required before any return is filed, ensuring that your professional judgment remains central to every compliance action.

Who is this for?

The CA Pack is designed for Chartered Accountant firms managing 5 to 50+ client companies. Whether you are a solo practitioner with a handful of clients or a mid-size firm with dozens, the platform scales to your needs. Typical users include:

  • Partners who need a bird's-eye view of compliance health across all clients
  • Managers who assign work and review agent outputs
  • Senior Associates who prepare and verify filings
  • Associates who handle day-to-day data entry and follow-ups
  • Audit Reviewers who need read-only access to the immutable audit trail

GSTIN and PAN Validation Rules

The system enforces strict validation on tax identifiers:

  • GSTIN: Must match the pattern NN AAAAA NNNN A X Z C where N=digit, A=alpha, X=alphanumeric. Example: 29AABCU9603R1ZM. The first two digits represent the state code.
  • PAN: Must match AAAAA NNNN A format. The 4th character indicates entity type (C=Company, P=Person, H=HUF, F=Firm, etc.). Example: AABCU9603R.

6. GST Compliance

How Auto-Filing Works (gst_auto_file = True)

When GST Auto-Filing is enabled for a client company, the GST Filing Agent follows this automated workflow:

1. Fetch GSTR-2A Automated
Agent fetches inward supply data from GSTN via the gstn:fetch_gstr2a tool.
2. Reconcile with Purchase Register Automated
Cross-references GSTR-2A data against the purchase register in Tally. Flags mismatches above 5% of return value.
3. Prepare GSTR-1 from Sales Register Automated
Generates GSTR-1 data from the sales register using tally:generate_gst_report.
4. Compute GSTR-3B Automated
Computes the GSTR-3B summary from GSTR-1 and GSTR-2A data.
5. Partner Review HITL Required
Partner reviews the prepared returns, tax liability summary, and either approves, requests changes, or rejects.
6. File GSTR-3B with DSC Automated
Agent files the return with DSC signature via gstn:file_gstr3b.
7. Notify Team Automated
Sends filing confirmation via email and Slack notification.
🛑 HITL Always Required

Even with auto-filing enabled, human-in-the-loop review is always required before any filing submission. The agent will not file a return without explicit partner approval. This is enforced at the workflow level and cannot be overridden.

How Manual Upload Works (gst_auto_file = False)

When auto-filing is disabled (the default), the workflow changes at the filing step:

  1. Agent generates the GSTR-1 and GSTR-3B data as JSON files.
  2. JSON files appear in the GSTN Manual Upload section of the Compliance tab.
  3. User clicks "Download" to save the JSON file locally.
  4. User logs into the GSTN portal and uploads the JSON manually.
  5. After successful upload, user returns to AgenticOrg and clicks "Mark as Uploaded".
  6. System prompts for the ARN (Acknowledgement Reference Number).
  7. Filing is recorded with the ARN for audit trail purposes.

GSTR-2A Reconciliation

The GST Filing Agent automatically reconciles GSTR-2A (inward supply) data against the client's purchase register. The reconciliation matches on:

  • Supplier GSTIN
  • Invoice number and date
  • Taxable value and tax amount

ITC Mismatch Detection

⚠️ 5% Threshold Rule

If the ITC (Input Tax Credit) mismatch exceeds 5% of the return value, the agent flags the return for manual review and will not proceed to filing. This prevents accidental over-claiming of ITC.

7. TDS Compliance

Overview

The TDS Compliance Agent automates the complete quarterly TDS lifecycle for each client company. It uses 7 tools and operates with GPT-4o for maximum accuracy on tax computations.

Form 26Q / 24Q Auto-Generation

1. Extract TDS-Applicable Transactions
Reads all transactions from Tally that require TDS deduction based on applicable sections.
2. Compute TDS Deductions per Section
Calculates TDS amounts per section (194A, 194C, 194H, 194J, etc.). Validates PAN and section applicability for every deductee.
3. Reconcile with 26AS
Cross-checks computed TDS against the 26AS statement. Ensures reconciliation within 1% tolerance.
4. Prepare Form 26Q (Non-Salary) & 24Q (Salary)
Generates both forms in parallel based on transaction type.
5. Pay TDS Challan
Prepares challan payment details via income_tax:pay_tax_challan.
6. Partner Review HITL Required
Partner reviews returns, challan details, and total TDS amount before approving.
7. File Returns & Generate Form 16A
Files Form 26Q and 24Q, then generates Form 16A certificates for all deductees.
8. Post Journal Entries to Tally
Posts TDS journal entries back to Tally via tally:post_voucher.

Form 16A Issuance

After quarterly returns are filed, the TDS agent automatically generates Form 16A certificates for all deductees using the income_tax:download_form_16a tool. Certificates are stored in the document vault.

26AS Reconciliation

ℹ️ 1% Tolerance

The TDS agent reconciles computed TDS against the 26AS statement. If the discrepancy exceeds 1% tolerance, the agent will not proceed and escalates to a senior associate for investigation.

8. Bank Reconciliation

Overview

The Bank Reconciliation Agent runs daily on weekdays at 9 AM IST for each client company. It achieves 99.7% auto-match accuracy using AI-powered matching on amount, date, and reference fields.

Account Aggregator Integration

Bank statements are fetched automatically via the Account Aggregator (AA) framework, eliminating the need for manual statement downloads. The agent uses two AA tools:

  • banking_aa:fetch_bank_statement — Fetches the daily bank statement
  • banking_aa:get_transaction_list — Retrieves individual transactions
  • banking_aa:check_account_balance — Verifies the closing balance

Auto-Match Process

1. Fetch Bank Statement
Retrieves yesterday's bank transactions via Account Aggregator.
2. Fetch Book Entries
Reads the corresponding book entries from Tally via tally:get_ledger_balance.
3. Auto-Match Transactions
AI engine matches bank transactions to book entries using amount, date, and reference. Confidence threshold: 0.95.
4. Flag Outstanding Items
Items unmatched for more than 30 days are flagged for review.
5. Check for Breaks
If break count > 0 or flagged items > 5, escalates to senior associate.
6. Generate Report
Produces a reconciliation report with matched/unmatched counts and closing balance.
7. Deliver Report
Sends the report via email to the manager and Slack to #bank-recon.

30-Day Stale Item Flagging

Any transaction that remains unmatched for more than 30 days is automatically flagged. If more than 5 items are flagged, the workflow escalates to a senior associate for investigation. The escalation provides a summary of the break count, break total amount, and a list of the flagged items.

Escalation to Partner

Reconciliation breaks are escalated through a human-in-the-loop step. The senior associate can choose to:

  • Resolve: Mark items as resolved with an explanation.
  • Investigate: Request further investigation before proceeding.
  • Defer: Defer resolution to the next reconciliation cycle.
ℹ️ Confidence Floor

The bank reconciliation agent uses a confidence floor of 0.95 (95%). Transactions matched with less than 95% confidence are flagged for manual review rather than being auto-posted. This prevents incorrect matching.

Matching Criteria

The auto-match engine uses a multi-factor scoring algorithm to match bank transactions with book entries:

FactorWeightDescription
Amount (exact match)40%Transaction amounts match exactly (to the paisa)
Date proximity25%Transaction dates within 3 business days of each other
Reference match20%UTR, cheque number, or narration contains matching reference
Counterparty match10%Payee/payer name matches the ledger party name
Transaction type5%Debit/credit direction matches

The combined score must exceed 0.95 (95%) for auto-matching. Scores between 0.75 and 0.95 are flagged as "probable matches" for human confirmation. Scores below 0.75 are marked as "unmatched" and require manual investigation.

Reconciliation Report

The daily reconciliation report includes:

  • Summary: Date, company name, bank name, account number (masked).
  • Match Statistics: Total transactions, matched count, unmatched count, match percentage.
  • Outstanding Items: List of items unmatched for >30 days with ageing categories (30-60d, 60-90d, 90d+).
  • Balance Reconciliation: Bank closing balance vs. book closing balance, with difference amount.
  • Action Items: List of items requiring human attention, sorted by amount (highest first).

9. Filing Approvals

How the Approval Workflow Works

Every filing (GST, TDS, or other statutory return) requires explicit approval before submission. The approval workflow follows this flow:

  1. An AI agent prepares the filing and creates an approval request.
  2. The request appears in the Approvals tab with status Pending.
  3. A partner reviews the request details (filing type, period, data summary).
  4. The partner either approves, rejects with reason, or requests changes.

10. GSTN Credential Vault

Why Encrypted Storage?

GSTN portal credentials are required for auto-upload functionality. The Credential Vault uses Fernet (AES-128-CBC) encryption at rest to ensure that credentials are never stored in plaintext. The encryption key is derived from the tenant's master key and is never transmitted to the client.

🔒 Security Guarantee

Credentials stored in the vault never appear in API responses, log files, or error messages. The API only returns the username and a masked indication that a password is stored. Decryption only occurs server-side during the filing process.

Adding Credentials

To add credentials to the vault, navigate to the Settings tab of the company detail view and scroll to the Credential Vault section.

20. Frequently Asked Questions

Q1: Can I use the CA Pack without Tally?

Yes. The Tally integration is optional. Without Tally, agents will rely on data from Zoho Books or manual uploads. However, full automation (including GST report generation and journal posting) requires Tally connectivity.

Q2: What happens if an agent makes an error?

All filing agents require human-in-the-loop (HITL) approval before submission. The agent will never file a return without your explicit approval. Additionally, agents have built-in validation rules (5% ITC threshold, 1% TDS tolerance) that prevent them from proceeding with suspicious data.

Q3: Can I add more than 50 client companies?

The standard plan supports up to 50 clients. For larger deployments, contact our support team at sanjeev@agenticorg.ai for enterprise pricing.

Q4: Is my data safe?

Yes. Data is encrypted at rest (AES-128), isolated via PostgreSQL Row-Level Security, and protected by SOC2-compliant audit trails. GSTN credentials are stored using Fernet encryption and never appear in API responses or log files.

Q5: What if the Tally Bridge goes offline?

If the bridge is unavailable, agents that depend on Tally data will fail gracefully and report a connectivity error. The workflow will retry automatically after a configurable interval. No data is lost.

Q6: Can different team members have different roles for different companies?

Yes. Roles are scoped per company. A user can be a Partner for Company A and a Senior Associate for Company B. This is managed in the Settings → Role Management section of each company.

Q7: How does the 14-day trial work?

The trial gives you full access to all CA Pack features for 14 days with a limit of 7 client companies. No credit card is required. After 14 days, the subscription expires and access becomes read-only until you upgrade.

Q8: Can I use both auto-filing and manual upload for different clients?

Yes. The gst_auto_file flag is configured per company. You can enable auto-filing for clients who have provided their DSC and GSTN credentials, and use manual upload for others.

Q9: How are compliance alerts sent?

Compliance alerts are sent via email to the compliance_alerts_email configured for each company. Two alerts are sent: a 7-day warning and a 1-day urgent alert before each deadline. The partner email also receives the 1-day urgent alerts.

Q10: What LLM models do the agents use?

The GST Filing Agent and TDS Compliance Agent use GPT-4o for maximum accuracy on tax computations. The Bank Reconciliation, FP&A, and AR Collections agents use GPT-4o-mini for cost-effective processing of high-volume tasks.

Q11: Can I export audit logs?

Yes. The audit log is accessible via the API at GET /api/v1/companies/:id/audit. Logs can be exported in JSON or CSV format. Logs are immutable, HMAC-signed, and retained for 7 years per SOC2 requirements.

Q12: Does AgenticOrg support GST composition scheme clients?

Yes. For composition scheme clients, the GST Filing Agent adjusts to prepare CMP-08 (quarterly statement) instead of monthly GSTR-1/3B. Configure this during onboarding or in the company settings.
